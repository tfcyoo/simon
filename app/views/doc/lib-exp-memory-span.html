Experimental Tasks

Memory Span

In the Memory Span paradigm, participants have to memorise a list of stimuli (e.g., digits in the case of a Digit Span, see available stimuli files below) and then recall them in correct serial order. In Tatool, the Memory Span has to be set as the primary Executable of a Dual Element. It can (but doesn't have to) be combined with the Choice Reaction Time task to create a Brown-Peterson or a Complex Span task.

Executable

Use the following settings in the Module Editor for this type of task:

Project tatool
Executable Type tatoolMemorySpan

The Memory Span Executable has to be used as the primary Executable of a Dual Element in order for it to work properly. Omit the secondary element to use the Memory Span Executable as a simple span task.

Properties

The following Properties can be provided in the Module Editor to change the behaviour of the task:

Property Name Property Type Default Mandatory Description
stimuliFile Resource - Yes A stimuli file expecting stimuli according to the Tatool Stimulus definition.
stimuliPath Path - Yes A stimuli path where to load any resources from (stimuli images, input key images).
suspendAfterEachItem Boolean true No Configures the Executable to suspend after each memorandum (true) as in the Complex Span paradigm, or after all memoranda (false) as in the Brown-Peterson paradigm.
displayDuration String 800 No Display duration for each memorandum.
intervalDuration String 400 No Interval duration between memoranda.
randomisation String (none) No Defines the randomisation criteria. If no value is set no randomisation is applied. The following values are allowed:
full: fully randomises stimuli (no duplicates until all stimuli are shown once)
recallText String Stimulus No Text to display in the recall phase. Text will always receive the number of the current recall stimulus as a suffix (e.g., 'Stimulus 1').

Stimuli File

This task requires a stimuli file to run. To allow set sizes / list lengths of n memoranda per trial, you can use the usual stimulus properties and add an index at the end (e.g., stimulusValue1). The stimuli file has to contain the columns specified in the Stimuli File Structure below.

Available Stimuli Files

Task Project Type File Name Description
Digit Span tatool-stimuli stimuli memorySpan-digits.csv Uses two-digit numbers as memoranda with different spans (3-5).

Stimuli File Structure

Column Name Column Values Mandatory Column Description
stimulusCount (String) Yes Set size / list length (i.e., number of memoranda) in this trial.
stimulusValue[n] (String)/(Image Filename) Yes Value to display on screen. Can either be a text or an image.
stimulusValueType[n] text/image/circle/square Yes The stimulus value type for Tatool to know how to display it.
correctResponse[n] (String) Yes The correct response for this memorandum.

Example Module

Download from GitHub: demoDigitSpan.json

The Example Module contains all elements of the demo and can be used to customize the task (for more details, see instructions for using your own stimuli and description of the Instruction Accessory Executable). Note that the Instruction Executable currently doesn't allow HTML pages from external resources. This means that you can use custom HTML pages if you host your own instance of Tatool Web, but you cannot use externally hosted HTML pages on tatool-web.com. For running experiments on tatool-web.com with customized instructions, you must use images (see the FAQ for currently available options to host your custom stimuli and instructions on tatool-web.com).
