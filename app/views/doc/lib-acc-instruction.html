Accessory Executables

Instruction

The primary purpose of this Executable is to display slides with instructions, but it can be used to present any arbitrary content using images or HTML files. It offers a pagination navigation allowing a user to either go to the next or previous slide, or to jump to a specific slide.

Note that the Instruction Executable currently doesn't allow HTML pages from external resources. This means that you can use custom HTML pages if you host your own instance of Tatool Web, but you cannot use externally hosted HTML pages on tatool-web.com. For running experiments on tatool-web.com with customized instructions, you must use images (see the FAQ for currently available options to host your custom stimuli and instructions on tatool-web.com).

Project tatool
Executable Type tatoolInstruction

Properties

The Instruction Executable can either draw from an array of HTML pages or an array of images, but not both. The following Properties can be provided in the Module Editor to change the behaviour of the task:

Property Name Property Type Default Mandatory Description
images Array (Resource) - (Yes) An array of project/external image resources (png, jpg, gif). The sequence of the items in the Array represents the display sequence.
pages Array (Resource) - (Yes) An array of project HTML resources. External HTML Resources are currently not supported. The sequence of the items in the Array represents the display sequence.
showDigitPagination Boolean true (No) Allows you to hide the digit buttons in the pagination navigation.

Stimuli File

This task does not support the use of a stimuli file.

HTML customisation

The Instruction Executable offers a set of template variables that can be embedded in your HTML instructions to allow for dynamic content. The following template variables are currently supported:

Variable Name Description
moduleName The name of the current Module.
moduleAuthor The author of the current Module.
moduleVersion The version of the current Module.
sessionNr The current Session ID.
moduleProperties JavaScript object representing all available moduleProperties. For example, to access the Module Property currentLevel of a level handler with the name level you would use moduleProperties.level.currentLevel.
sessionProperties JavaScript object representing all available sessionProperties. For example, to access the Session Property myTrialNo of an Executable with the name myExecutable you would use sessionProperties.myExecutable.myTrialNo.

Example HTML

<div id="tatoolFormat">

  <h1>Welcome to {{moduleName}}</h1>
  
  <p>
    <center>
      You're currently in Session <span class='highlight_red'>{{sessionNr}}</span> of 10 Sessions.
    </center>
  </p>

  <img ng-src="welcome.gif">

</div>
